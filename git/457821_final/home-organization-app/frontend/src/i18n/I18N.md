# i18n structure

## Locales

- `locales/en.json`, `he.json`, `ru.json` — full base packs.
- `locales/fr.overrides.json`, `ar.overrides.json` — merged **on top of English** via `mergeDeep` (missing keys fall back to `en`).

## Namespaces (`localeBundle.ts`)

| Namespace        | Source in JSON (legacy `ui` still supported) |
|-----------------|-----------------------------------------------|
| `layout`        | `layout` **or** `ui.layout`                   |
| `room`          | `room` **or** `ui.roomPage`                   |
| `visionBoard`   | `visionBoard` **or** `ui.visionBoard`       |
| `settingsUi`    | `settingsUi` **or** `ui.settingsPage`         |
| `rooms` + `hub` | `rooms` + merge of `rooms.hub` and `ui.roomsPage` |
| `emotionalJournal` | top-level `emotionalJournal`            |
| `emptyStates`   | top-level `emptyStates`                       |
| `dashboard` / `challenge` | reserved for future Dashboard refactor |

Avoid adding new strings under a monolithic `ui` in new work — prefer the domains above.

## RTL

Use `isRtlLang(i18n.language)` from `utils/localeDirection.ts` (`he`, `ar` → RTL).

## Room display

Use `getLocalizedRoomTitle(name, tRooms, { roomId })` from `utils/roomLocalization.ts`.  
Canonical keys are resolved via `resolveRoomKey` + `rooms.room_types.*`; unknown names use `rooms.hub.fallbackRoomId`.
